Проведи code review текущих изменений.

Шаги:
1. Запусти `git diff` и изучи изменения
2. Проверь соответствие конвенциям из CLAUDE.md
3. Если есть `docs/standards/` — свери с применимыми стандартами
4. Для firmware: проверь модульность, отсутствие глобального состояния, корректность SPI/UART операций
5. Для software: проверь разделение core/ и ui/, thread-safety SerialReader, корректность numpy-операций
6. Проверь правило языка: код, комментарии — на английском
7. Дай резюме на русском: что хорошо, что исправить (файл:строка)

Будь конкретным. Не переписывай код без запроса — только предлагай.
